Dentro da VM, instala Docker e Docker Compose:

sudo apt update
sudo apt install -y docker.io docker-compose
sudo systemctl enable --now docker
sudo usermod -aG docker $USER


Fecha a sessão e abre de novo (ou faz su - $USER) para o teu usuário ter permissão de usar Docker sem sudo.

2️⃣ Cria a estrutura de pastas
mkdir -p ~/inception/{nginx,wordpress,mariadb,volumes,compose}
cd ~/inception


nginx → configuração do Nginx com TLS

wordpress → WordPress + php-fpm

mariadb → MariaDB

volumes → arquivos persistentes do WordPress

compose → docker-compose.yml


3️⃣ Cria os volumes
docker volume create wordpress_db
docker volume create wordpress_files


wordpress_db → vai armazenar a base de dados

wordpress_files → vai armazenar os arquivos do WordPress



4️⃣ Cria a rede Docker
docker network create inception-network


Todos os containers vão se comunicar usando essa rede.

5️⃣ Container MariaDB

Cria um Dockerfile simples opcional, mas podes usar diretamente a imagem oficial:

docker run -d \
  --name mariadb \
  --network inception-network \
  -e MYSQL_ROOT_PASSWORD=senha_root \
  -e MYSQL_DATABASE=wordpress \
  -e MYSQL_USER=wp_user \
  -e MYSQL_PASSWORD=senha_wp \
  -v wordpress_db:/var/lib/mysql \
  mariadb:latest

6️⃣ Container WordPress (php-fpm)

Apenas o php-fpm, sem Nginx:

docker run -d \
  --name wordpress \
  --network inception-network \
  -e WORDPRESS_DB_HOST=mariadb:3306 \
  -e WORDPRESS_DB_USER=wp_user \
  -e WORDPRESS_DB_PASSWORD=senha_wp \
  -e WORDPRESS_DB_NAME=wordpress \
  -v wordpress_files:/var/www/html \
  wordpress:php8.1-fpm


mariadb é o nome do container da MariaDB na mesma rede, usado como host DB.

7️⃣ Container Nginx com TLS

Cria um nginx.conf no diretório nginx:

server {
    listen 443 ssl;
    server_name localhost;

    ssl_certificate /etc/ssl/certs/cert.pem;
    ssl_certificate_key /etc/ssl/private/key.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    root /var/www/html;
    index index.php index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    location ~ \.php$ {
        fastcgi_pass wordpress:9000;
        fastcgi_index index.php;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    }
}


Cria certificados autoassinados para teste:

mkdir -p ~/inception/nginx/ssl
openssl req -x509 -nodes -days 365 \
  -newkey rsa:2048 \
  -keyout ~/inception/nginx/ssl/key.pem \
  -out ~/inception/nginx/ssl/cert.pem \
  -subj "/CN=localhost"


Rodar o container Nginx:

docker run -d \
  --name nginx \
  --network inception-network \
  -p 443:443 \
  -v wordpress_files:/var/www/html \
  -v ~/inception/nginx/nginx.conf:/etc/nginx/conf.d/default.conf:ro \
  -v ~/inception/nginx/ssl:/etc/ssl \
  nginx:latest

8️⃣ Resumo da arquitetura
[nginx container] <--HTTPS--> Browser
       |
       | fastcgi_pass
       v
[wordpress container] <--MySQL--> [mariadb container]


Rede inception-network conecta todos os containers.

Volumes persistem dados (wordpress_db e wordpress_files).

TLS configurado no Nginx com apenas TLSv1.2 e TLSv1.3.
